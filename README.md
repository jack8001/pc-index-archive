# pc-index-archive

## 项目简介

`pc-index-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为通用页面存档与分发用途，方便历史页面或离线展示内容的整理与访问。

## 目录说明

```
pc-index-archive/
├── index.html          # 主入口页面，用于索引其他归档页面
├── pages/              # 存放各独立 HTML 页面文件
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 页面依赖的静态资源（CSS、JS、图片等）
│   ├── styles/
│   ├── scripts/
│   └── images/
└── README.md           # 本说明文件
```

- `index.html`：仓库的主导航页，可在此链接到 `pages/` 下的各页面。
- `pages/`：所有归档的 HTML 页面均放置于此目录下，文件名应保持唯一且具有描述性。
- `assets/`：存放页面所需的公共资源文件，按类型分目录管理。

## 页面归档说明

- 每个归档页面均为独立的 HTML 文件，可脱离仓库环境直接打开运行。
- 页面内容应遵循通用 Web 标准，避免依赖外部服务或非公开资源。
- 如需引用仓库内资源，请使用相对路径（如 `../assets/styles/main.css`）。
- 建议在页面 `<head>` 中添加简短的注释说明归档时间或来源版本。

## 维护说明

- 本仓库主要面向个人或团队内部使用，也可作为公开存档。
- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并在 `index.html` 中添加对应链接。
- 更新或替换页面时，建议保留旧版本文件（可添加后缀如 `_v1`），或使用 Git 历史记录管理。
- 不鼓励在仓库中包含任何动态后端逻辑或敏感信息。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任意页面。
3. 可通过 GitHub Pages 或其他静态托管服务部署本仓库，实现在线浏览。
